# LSMD Supabase Database

Аналітична база даних ЛСМД (Лікарня Швидкої Медичної Допомоги, Чернівці).

## Технологія
- **Платформа:** Supabase
- **Регіон:** eu-west-1
- **PostgreSQL:** 17

## Структура (26 таблиць)

### Клінічні таблиці
- `lsmd` (110 206) — журнал випадків
- `patients_best` (72 293) — пацієнти
- `empl` (868) — працівники
- `lsmd_doctors` (282) — міст lsmd ↔ empl

### Штатна структура
- `departments` (20) — відділення з head_empl_id, staff_count
- `dept_categories` (2) — клінічне / не_клінічне
- `clinical_blocks` (3) — хірургічний / терапевтичний / інтенсивна_терапія
- `dept_transfers_matrix` (143)

### Чергування
- `doctor_shifts` (68 299) — реконструйовані 09:00→09:00
- `doctor_stats` (282) — агрегати з нічними/вихідними

### МКХ-10
- `icd_chapters` (22) — глави
- `icd_blocks` (220) — підкатегорії
- `icd_10` (19 824) — діагнози
- 17 малих довідників

### Операції
- `operations` (7 320) — комбінації ACHI-кодів

## VIEW
- `interventions` — оперативні втручання
- `lsmd_transfers` — переведення
- `lsmd_deaths_24h` — смерті в першу добу
- `doctor_patient_links` — зв'язок пацієнт-лікар
- `doctor_discharges` — виписки лікаря
- `doctor_diagnoses` — діагнози лікаря
- `lsmd_shift_type` — класифікація змін

## Безпека
⚠️ В репо НЕ зберігаються:
- API ключі Supabase (anon, service_role)
- Конкретні дані пацієнтів
- .env файли з паролями
